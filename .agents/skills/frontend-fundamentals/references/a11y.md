# 접근성 (A11y) 상세 가이드

출처: https://frontend-fundamentals.com/a11y/

## 접근성이란

접근성(Accessibility, A11y)은 모든 사용자가 더 쉽고 편리하게 웹을 사용할 수 있도록 돕는 기본 원칙이다. 프론트엔드 개발자는 HTML 구조를 만들고 인터랙션을 정의하므로 **접근성의 출발점이자 핵심 역할**을 맡고 있다.

---

## 스크린 리더의 3요소

스크린 리더는 화면 속 요소와 정보를 음성으로 전달하는 보조 기술이다. 3가지 핵심 정보를 읽는다:

1. **역할(Role)**: 요소가 어떤 종류인지 (예: 버튼, 입력창, 스위치)
2. **레이블(Label)**: 컴포넌트의 이름, 어떤 기능인지 설명
3. **상태(State)**: 현재 상태 (예: 활성화됨, 꺼짐, 선택됨)

### 읽는 순서 예시
```html
<div role="radio" tabindex="0" aria-checked="true" aria-label="마케팅 알림"></div>
```
스크린 리더가 읽는 순서:
1. "마케팅 알림" (레이블)
2. "체크상자" (역할)
3. "선택됨" (상태)
4. "설정을 끄거나 켜려면 이중탭 하십시오" (자동 설명)

### 탭 목록 예시
```tsx
<div role="tablist" aria-label="메뉴">
  <button role="tab" aria-selected={true}>첫번째 항목</button>
  <button role="tab" aria-selected={false}>두번째 항목</button>
</div>
```
스크린 리더: "메뉴, 탭그룹 → 첫번째 항목, 탭, 선택됨 → 두번째 항목, 탭"

## 접근성 개발 입문: 역할, 레이블, 상태

### 역할 (Role)
HTML 시맨틱 태그를 사용하면 역할이 자동으로 부여된다:
- `<button>` → role="button"
- `<input type="checkbox">` → role="checkbox"
- `<nav>` → role="navigation"

시맨틱 태그가 없을 때는 `role` 속성으로 명시:
```html
<div role="tablist">...</div>
<div role="tab">...</div>
<div role="tabpanel">...</div>
```

### 레이블 (Label)
인터랙티브 요소에는 반드시 접근 가능한 이름이 필요하다.

방법 1: 텍스트 콘텐츠
```html
<button>저장하기</button>
```

방법 2: `aria-label`
```html
<button aria-label="닫기"><CloseIcon /></button>
```

방법 3: `aria-labelledby`
```html
<h2 id="section-title">알림 설정</h2>
<div role="region" aria-labelledby="section-title">...</div>
```

### 상태 (State)
동적으로 변하는 요소의 현재 상태를 전달한다:
- `aria-checked`: 체크박스/라디오/스위치의 선택 여부
- `aria-selected`: 탭/옵션의 선택 여부
- `aria-expanded`: 아코디언/드롭다운의 펼침 여부
- `aria-disabled`: 비활성화 여부
- `aria-hidden`: 스크린 리더에서 숨김 여부

## 4가지 핵심 원칙

### 원칙 1: 올바른 구조 만들기

#### 버튼 안에 버튼 넣지 않기
HTML 명세상 `<button>` 안에 `<button>`을 넣으면 안 된다. 인터랙티브 요소 중첩 금지.

**문제 코드**:
```html
<button>
  카드 내용
  <button>삭제</button>  <!-- 금지! -->
</button>
```

**해결**: 카드 전체를 `<div>`로 만들고 내부 버튼만 `<button>`으로 처리하거나, 카드 클릭 영역과 버튼 영역을 분리한다.

#### 테이블 행에 직접 onClick 붙이지 않기
`<tr>` 요소에 `onClick`을 직접 붙이면 키보드 사용자가 접근할 수 없다.

**해결**: 행 안에 링크(`<a>`)나 버튼(`<button>`)을 두어 키보드 포커스가 가능하게 한다.

### 원칙 2: 의미를 정확히 전달하기

#### 인터랙티브 요소에 이름 붙이기
아이콘만 있는 버튼, 입력 필드 등에는 반드시 접근 가능한 이름을 제공한다.

**Bad**:
```html
<button><SearchIcon /></button>  <!-- 이름 없음! -->
```

**Good**:
```html
<button aria-label="검색"><SearchIcon /></button>
```

#### 같은 이름의 요소에는 설명 추가하기
여러 "더보기" 버튼이 있으면 어떤 "더보기"인지 구별할 수 없다. `aria-describedby` 또는 더 구체적인 `aria-label`을 사용한다.

### 원칙 3: 예측 가능한 동작 만들기

#### 버튼의 역할과 동작이 일치하게 만들기
`<div onClick={...}>`으로 버튼을 흉내 내면 안 된다. 키보드 이벤트(`Enter`, `Space`)가 작동하지 않고, 스크린 리더가 버튼으로 인식하지 못한다.

**Bad**:
```html
<div onClick={handleClick} style={{ cursor: 'pointer' }}>클릭</div>
```

**Good**:
```html
<button onClick={handleClick}>클릭</button>
```

#### 입력 요소는 `<form>`으로 감싸기
`<form>` 태그로 감싸면 Enter 키로 제출, 자동완성, 스크린 리더의 폼 모드 등이 자동으로 작동한다.

### 원칙 4: 시각 정보 보완하기

#### 이미지와 아이콘에 대체 텍스트 제공하기
- 정보를 전달하는 이미지: `alt` 텍스트 필수
- 장식적 이미지: `alt=""` (빈 문자열) 또는 `aria-hidden="true"`
- 아이콘 버튼: `aria-label` 사용

```html
<!-- 정보 이미지 -->
<img src="chart.png" alt="2024년 매출 추이 그래프" />

<!-- 장식 이미지 -->
<img src="decoration.png" alt="" />

<!-- 아이콘 버튼 -->
<button aria-label="설정"><GearIcon aria-hidden="true" /></button>
```

## UI 요소별 접근성 기초

### 탭 (Tab)
```tsx
<div role="tablist" aria-label="설정 메뉴">
  <button role="tab" aria-selected={true} aria-controls="panel-1">일반</button>
  <button role="tab" aria-selected={false} aria-controls="panel-2">보안</button>
</div>
<div role="tabpanel" id="panel-1">일반 설정 내용</div>
```
키보드: 좌우 화살표로 탭 이동, Enter/Space로 선택

### 아코디언 (Accordion)
```html
<h3>
  <button aria-expanded="true" aria-controls="section1">섹션 제목</button>
</h3>
<div id="section1" role="region">내용</div>
```

### 모달 (Modal)
- `role="dialog"` 또는 `role="alertdialog"`
- `aria-modal="true"`
- 포커스 트랩: 모달 내부에서만 Tab 이동
- ESC로 닫기
- 열릴 때 첫 번째 포커스 가능 요소로 포커스 이동
- 닫힐 때 트리거 요소로 포커스 복원

### 라디오 (Radio)
```html
<div role="radiogroup" aria-label="결제 방식">
  <div role="radio" aria-checked="true" tabindex="0">카드</div>
  <div role="radio" aria-checked="false" tabindex="-1">계좌이체</div>
</div>
```
키보드: 화살표로 이동, 선택과 포커스가 동시에 이동

### 체크박스 (Checkbox)
- `aria-checked`: `"true"`, `"false"`, `"mixed"` (부분 선택)
- Space로 토글

### 스위치 (Switch)
```html
<button role="switch" aria-checked="true" aria-label="알림">ON</button>
```

## 심화: ESLint로 접근성 개선하기

### 주요 규칙
`eslint-plugin-jsx-a11y` 플러그인의 주요 규칙:
- `alt-text`: `<img>`에 `alt` 속성 필수
- `anchor-is-valid`: `<a>`에 유효한 `href` 필수
- `click-events-have-key-events`: `onClick`에 키보드 이벤트 동반
- `no-static-element-interactions`: 비인터랙티브 요소에 이벤트 핸들러 금지
- `label-has-associated-control`: `<label>`과 폼 컨트롤 연결

### 디자인 시스템과 결합하기
디자인 시스템의 컴포넌트에 접근성 속성을 필수로 요구하는 커스텀 ESLint 규칙을 만들면, 팀 전체의 접근성 수준을 일관되게 유지할 수 있다.
